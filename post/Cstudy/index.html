C++学习笔记 —— 函数

  1. 2020-03-22
  2. 4849字
  3. 18 min read

函数

分类：系统函数、自定义函数、有参函数、无参函数

返回值

  • C++要求在定义函数时必须指定函数的类型，如无返回值需指定void
  • 一个函数中可以有多个return语句
  • return后面的括号可以省略，即return z;与return(z)等价
  • return的类型要和函数的类型一致
  • void表示函数没有返回值或没有参数
  • 在C语言的函数定义中，如果省略了return语句，函数也可以返回主调函数（void）
  • 不能返回自动变量的指针
    int *f(void)				//f是一个无参数的函数，返回的是一个指向整型数据的指针
    { int i;
    return &i;}				//函数结束时局部变量i被释放空间

被调函数运行结束后系统为被调函数中的局部变量分配的内存空间就会被释放。
在执行 return 语句时系统是在内部自动创建了一个临时变量，然后将 return 要返回的那个值赋给这个临时变量。所以当被调函数运行结束后 return 后面的返回值就被释放掉了，最后是通过这个临时变量将值返回给主调函数的。而且定义函数时指定的返回值类型实际上指定的就是这个临时变量的类型。
这也是为什么当 return 语句中表达式的类型和函数返回值类型不一致时，将 return 的类型转换成函数返回值类型的原因。return 语句实际上就是将其后的值赋给临时变量，所以它要以临时变量的类型为准，即函数返回值的类型。

函数调用

  • main函数由系统调用
  • 所有函数是平行的，不能嵌套定义
  • 各函数之间可以互相调用，但不能调用main函数
  • 如果main函数在其他函数前进行定义，在调用该函数时需进行声明
  • 调用函数时，要给出参数。在主调函数和被调函数之间有数据传递。即主调函数将数据传给被调函数使用。被调函数可以带回或者不带回（此时函数类型为void）函数值。

形参 & 实参

  • 形式参数：在定义函数时函数名后面括号中的变量名称
  • 实际参数：在调用一个函数时，函数名后面括号中的参数，可以是常量、变量或表达式
  1. 实参变量对形参变量的数据传递是“值传递”，即单向传递，只由实参传给形参
  2. 实参与形参的类型可以不相同（发生强制转换）
  3. 形参只有在函数调用时才被分配内存单元，以便接收从实参传来的数据。在调用结束后，形参所占的内存单元也被释放。因此，在执行一个被调用函数时，形参的值如果发生改变，并不会改变主调函数中实参的值。
  4. 实参单元与形参单元是不同的单元，实参形参占用不同的空间。

内置函数 / 内联函数

C++提供一种提高效率的方法，即在编译时将所调用函数的代码直接嵌入到主调函数中，编译系统在遇到该函数调用时，就用该函数体的代码进行代替，同时将实参代替形参。

  • 可以在声明函数和定义函数时同时写inline，也可以只在函数声明时加inline，效果相同。
  • 使用内置函数可以节省运行时间，但增加了目标程序的长度。因此一般只将规模很小而使用频繁的函数声明为内置函数。
  • inline声明只是建议性的，而不是指令性的。
    函数调用时需要一定的时间和空间的开销：保护现场、恢复现场
    一般的程序调用函数时的过程：
  1. 执行函数中调用前的语句；
  2. 转移到被调用函数的入口处，同时进行参数传递；
  3. 执行被调用的函数；
  4. 带回被调用函数的返回值；
  5. 接着执行主调函数中未执行的语句。

函数的重载（运算符重载）

C++允许用同一函数名定义多个函数

  • 重载函数的参数个数、参数类型或参数顺序三者中必须至少有一种不同
  • 函数的返回值类型可以相同也可以不同，不要求函数体相同

函数模板

建立一个通用函数，其函数类型和形参类型不具体指定，用一个虚拟的类型来代表。
template < typename T> template
通用函数定义 通用类定义
  • 函数体相同的函数都可以用这个模板来代替，不必定义多个函数，只需在模板中定义一次
  • 在调用函数时系统会根据实参的类型来取代模板中的虚拟类型
  • 类型参数可以不只一个，可以根据需要确定个数。
  • 只适用于函数的参数个数相同而类型不同，且函数体相同的情况


    编译器由模板自动生成函数时，会用具体的类型名对模板中所有的类型参数进行替换，其他部分原封不动地保留。同一个类型参数只能替换为同一种类型。编译器在编译到调用函数模板的语句时，会根据实参的类型判断如何替换模板中的类型参数。T 是类型参数，代表类型
#include <iostream>
using namespace std;
template<class T>
void Swap(T & x, T & y)
{
    T tmp = x;
    x = y;
    y = tmp;
}
int main( )
{
    int n = 1, m = 2;
    Swap(n, m);     //编译器自动生成void Swap(int &,int &)函数
    double f = 1.2, g = 2.3;
    Swap(f, g);     //编译器自动生成void Swap(double &,double &)函数
    return 0;
}

有默认参数的函数

  • 实参与形参结合顺序为从左至右，因此，指定默认值的参数必须放在形参表列中的最右端
  • 如果函数的定义在函数调用之前，则应在函数定义中给出默认值。
  • 一个函数不能既作为重载函数，又作为有默认参数的函数。

作用域

每一个变量都有其有效作用范围，称为变量的作用域。
在作用域以外是不能访问这些变量的。

局部变量 / 内部变量

保存在栈中，所在函数被调用时才由系统动态在栈中分配临时性的存储单元。

  • 在复合语句中定义的变量只在本复合语句范围内有效
  • 主函数中定义的变量也只在主函数中有效
  • 不同函数中可以使用同名的变量,它们代表不同的对象,互不干扰。
  • 形式参数也是局部变量。
  • 在函数声明中出现的参数名，其作用范围只在本行的括号内。

全局变量 / 外部变量

在函数之外定义的变量。保存在内存的全局存储区中，占用静态的存储单元。
有效范围为从定义变量的位置开始到本源文件结束。

  • 全局变量在程序的全部执行过程中都占用存储单元，而不是仅在需要时才开辟单元。
  • 它使函数的通用性降低了，因为在执行函数时要受到外部变量的影响。
  • 使用全局变量过多，会降低程序的清晰性。
    如果在同一个源文件中，全局变量与局部变量同名，则在局部变量的作用范围内，全局变量被屏蔽，即不起作用。要用this指定为全局变量。

C语言内存区

栈（stack）

由编译器进行管理，自动分配和释放，存放函数调用过程中的各种参数、局部变量、返回值以及函数返回地址。

  1. 申请后，只要栈的剩余空间足够，系统将为程序提供内存，否则异常提示栈溢
    出overflow。
  2. 栈是向低地址扩展的数据结构，是一块连续的内存区域(栈顶的地址和栈的最大容量是系统预先规定好的)，大小是2MB。 —> 栈获得的空间较小
  3. 堆获得的空间比较灵活，也比较大
  4. 在函数调用时，第一个进栈的是主函数中后的下一条指令（函数调用语句的下一条可
    执行语句）的地址，然后是函数的各个参数，在大多数的C编译器中，参数是由右往左入栈
    的，然后是函数中的局部变量。注意静态变量是不入栈的。
    当本次函数调用结束后，局部变量先出栈，然后是参数，最后栈顶指针指向最开始存的地
    址，也就是主函数中的下一条指令，程序由该点继续运行。
  5. 操作方式类似数据结构中的栈。

堆（heap）

用于程序动态申请分配和释放空间。在C中用malloc，C++中用new。需要程序员自己申请，并指明大小，申请的空间在使用结束后应该释放，若程序员没有释放空间，则程序结束时系统自动回收。

  1. 申请后，操作系统会 1.遍历一个记录空闲内存地址的链表，寻找第一个空间大于所申请空间的堆结点，2.然后将该结点从空闲结点链表中删除，并将该结点的空间分配给程序，3.同时在这块内存空间中的首地址处记录本次分配的大小，这样，代码中的delete语句才能正确的释放本内存空间。4.另外，由于找到的堆结点的大小不一定正好等于申请的大小，系统会自动的将多余的那部分重新放入空闲链表中。
  2. 堆是向高地址扩展的数据结构，是不连续的内存区域(系统是用链表来存储
    的空闲内存地址的)，堆的大小受限于计算机的虚拟内存。—> 堆获得的空间比较灵活，也比较大
  3. 堆是由new分配的内存，一般速度比较慢，而且容易产生内存碎片，但是方便
  4. 一般是在堆的头部用一个字节存放堆的大小。堆中的具体内容由程序员安排。
  5. 注意：这里的“堆”并不是数据结构中的“堆”。
void main()
{
    char a = 1;
    char c[] = "1234567890";    //栈上数组：在运行时刻赋值
    //在读取时直接就把字符串中的元素读到寄存器cl中
    char *p ="1234567890";      //指针所指向的字符串(例如堆)：在编译时就确定
    //要先把指针值读到edx中，再根据edx读取字符，更慢
    a = c[1];
    a = p[1];
return;
}

全局（静态）存储区

全局初始化区（DATA段）

存放已经初始化的全局变量和静态局部变量(static修饰的已经初始化的变量)
这些变量需要占用存储器的空间，在程序执行时它们需要位于可读写的内存区域内，并且有初值，以供程序运行时读写。

未初始化段（BSS段）

存放未初始化的全局变量和静态变量。（在程序中声明，但未初始化，在程序运行前不占用存储器空间）
可读写，在程序运行结束时自动释放。BBS段在程序执行之前会被自动清0，所以未初始化的全局变量和静态变量在程序执行之前已经为0。

文字常量区

存放常量字符串。程序结束后由系统释放。
char *name = "fengke"; 保存在文字常量区
char name1[] = "fengke" 保存在栈区

char *name = "fengke";
name 0x004156b8 —> unsigned char *  //name的值是保存在文字常量区"fengke"的地址
&name 0x0013ff5c —> unsigned char * *    //name自身的地址则被编译器分配在栈区
name[0] 102 —> unsigned char      //name的第一个字符值为'f'
&name[0] 0x004156b8 —> unsigned char *   //它保存在文字常量区
  • &name与name的地址不同 / 字符串常量的指针指向的内容和指针自身的地址不同
    因为name是个字符型变量，该变量的空间分配在栈区，name指向的字符串在文字常量区，而name又是个字符型指针变量，它里面所保存的值即为字符串在文字常量区的地址

程序代码区：存放程序的二进制代码。

存储空间

可以分为三部分：程序区、静态存储区、动态存储区

动态存储区 （ 栈 / 堆 ）

形参、自动变量（未加static声明的局部变量）、函数调用的现场保护和返回地址
在函数调用开始时分配动态存储空间，函数结束时释放这些空间。
这种分配和释放是动态的在程序运行期间，系统对变量动态地分配存储空间

静态存储区

静态变量、全局变量
在程序开始执行时给变量分配存储单元，程序执行完毕就释放这些空间。
在程序执行过程中它们占据固定的存储单元，而不是动态地进行分配和释放。

存储类别

自动的(auto)

  • 数据存储在动态存储区中，在函数调用结束时就自动释放这些空间
  • 关键字auto可以省略，如果不写auto，则系统把它默认为自动存储类别。
  • 在函数调用时赋初值，每调用一次函数重新给一次初值。
    如果不赋初值，则它的值是一个不确定的值。

静态的(static)

  • 在静态存储区内分配存储单元。在程序整个运行期间都不释放。
  • 赋初值是在编译时进行的，即只赋初值一次。以后每次调用函数时不再重新赋初值而只是保留上次函数调用结束时的值。
    如果在定义时不赋初值的话，编译时自动赋初值0(对数值型变量)或空字符(对字符型变量)。
  • 静态存储要多占内存，而且降低了程序的可读性
  • 虽然静态局部变量在程序生成时，在函数调用结束后仍然存在，但其他函数是不能引用它的。
int f(int n) {
    static int k, s;
    n--;
    for(k=n; k>0; k--)
		s += k;  
    return s;
}
int main(void)
{  	int k;
    k=f(3);
    printf("(%d,%d)", k, f(k));        //输出(3,6)
}

寄存器的(register)

  • 将局部变量的值放在CPU中的寄存器中，需要用时直接从寄存器取出参加运算，不必再到内存中去存取。
  • 只是建议性(而不是强制性)的

外部的(extern)

在一个文件内声明全局变量

  • 提前引用声明
    如果在定义点之前的函数想引用该全局变量，则应该在引用之前用关键字extern对该变量作外部变量声明，表示该变量是一个将在下面定义的全局变量。有了此声明，就可以从声明处起，合法地引用该全局变量
  • 在任一个文件中定义外部变量，而在另一文件中用extern对其作外部变量声明：

  • 静态外部变量(static)
    加上static声明，只能用于本文件的外部变量（全局变量）

变量属性

存储类型、作用域、数据类型、存储期

从作用域角度分

  • 局部变量：
    1. 自动变量,即动态局部变量(离开函数,值就消失)
    2. 静态局部变量(离开函数,值仍保留)
    3. 寄存器变量(离开函数,值就消失)
    4. 形式参数(可以定义为自动变量或寄存器变量)
  • 全局变量
    1. 静态外部变量(只限本文件引用)
    2. 外部变量(即非静态的外部变量,允许其他文件引用)

从变量存储期(存在的时间)来区分

  • 动态存储
    1. 自动变量(本函数内有效)
    2. 寄存器变量(本函数内有效)
    3. 形式参数
  • 静态存储
    1. 静态局部变量(函数内有效)
    2. 静态外部变量(本文件内有效)
    3. 外部变量(其他文件可引用)

从变量值存放的位置来区分

  • 内存中静态存储区
    1. 静态局部变量
    2. 静态外部变量(函数外部静态变量)
    3. 外部变量(可为其他文件引用)
  • 内存中动态存储区： 自动变量和形式参数
  • CPU 中的寄存器： 寄存器变量

声明&定义

函数：函数的原型是声明，函数功能的确立是定义
变量：建立存储空间的是定义，不需要建立存储空间的是声明
在一个程序中，变量只能定义一次，却可以声明多次。

  • 定义分配存储空间，而声明不会。
    extern 是声明不是定义,除非有extern关键字，否则都是变量的定义。
    通过使用extern关键字声明变量名而不定义它。

例如：extern int a 其中变量 a 可以在别的文件中定义的。
当你要引用一个全局变量的时候，你就要声明 extern int a , 省略了，就变成 int a 这是一个定义，不是声明。
（在函数外int定义变量后，在函数内extern int 声明为外部变量）

extern int i;       //声明，不是定义
int i;              //声明，也是定义
extern int a;       // 声明一个全局变量 a
int a;              // 定义一个全局变量 a
extern int a =0;    // 定义一个全局变量 a 并给初值。
                    //一旦给予赋值，一定是定义，定义才会分配存储空间
int a =0;           //定义一个全局变量 a，并给初值，等于第三个，都是定义一个可以被外部使用的全局变量，并给初值。

函数原型 / 函数声明 (function prototype)

一般形式为

  1. 函数类型 函数名(参数类型1，参数类型2…)；
  2. 函数类型 函数名(参数类型1 参数名1，参数类型2 参数名2…)；
    根据函数原型在程序编译阶段对调用函数的合法性进行全面检查。如果发现与函数原型不匹配的函数调用就报告编译出错。它属于语法错误。

内部函数（静态函数）

只能被本文件中其他函数所调用的函数，使函数只局限于所在文件
static int fun(int a,int b)

外部函数

在函数首部的最左端冠以关键字extern，可供其他文件调用
extern int fun (int a, int b)
如果在定义函数时省略extern,则默认为外部函数
在需要调用此函数的文件中,用extern声明所用的函数是外部函数。
